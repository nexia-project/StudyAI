## 2. FONTES — PAINEL ESQUERDO

Upload e importação dos seguintes tipos de fonte:
- PDF (extrair texto com pdf-parse, preservar páginas)
- Documentos Word .docx (parsing com mammoth)
- Arquivos de texto .txt e .md
- Links de páginas web (fetch + extrair texto limpo do HTML)
- Vídeos do YouTube (extrair transcrição/legendas com timestamps)
- Áudios MP3/WAV/M4A (transcrição com OpenAI Whisper API)
- Imagens PNG/JPG (OCR com Tesseract.js)
- Planilhas CSV/XLSX (parsing para texto estruturado)
- Apresentações .pptx (extração de texto dos slides)
- Google Docs (via link público)
- Google Sheets (via link público)
- EPUB (e-books)

Para cada fonte:
- Mostrar nome, tipo (ícone), tamanho, data de upload
- Preview do conteúdo ao clicar
- Botão de deletar
- Indicador de status: processando / pronto / erro

Limites: até 50 fontes por caderno, 200MB por arquivo, 500.000 palavras por fonte.

Drag & drop para upload de múltiplos arquivos simultaneamente.

## 3. PIPELINE RAG (BACKEND)

Quando uma fonte é adicionada, executar automaticamente:

a) EXTRAÇÃO: Extrair todo o texto do documento conforme o tipo
b) CHUNKING: Dividir em chunks de ~500 tokens com overlap de 50 tokens. Cada chunk guarda: texto, sourceId, pageNumber/posição
c) EMBEDDINGS: Gerar embedding de cada chunk usando OpenAI text-embedding-3-small (1536 dimensões)
d) ARMAZENAMENTO: Salvar embeddings no PostgreSQL com pgvector (criar extensão se não existir)

Quando o usuário faz uma pergunta no chat:
a) Gerar embedding da pergunta
b) Busca vetorial (cosine similarity) nos chunks do caderno — retornar top 15 mais relevantes
c) Montar contexto com os chunks recuperados, cada um identificado: [Fonte: nome_do_arquivo, Página X]
d) Enviar para GPT-4o com system prompt:

"Você é um assistente de pesquisa do Study.IA. Responda EXCLUSIVAMENTE com base no contexto fornecido abaixo. Se a informação não estiver no contexto, diga claramente que não encontrou nas fontes do caderno. SEMPRE cite suas fontes usando numeração [1], [2], [3] etc. Seja preciso e útil."

e) Fazer streaming da resposta (Server-Sent Events)

## 6. STUDIO — PAINEL DIREITO (Geração de Conteúdo)

Cada item abaixo é um botão no painel. Ao clicar, a IA processa TODAS as fontes do caderno e gera o conteúdo. Cada output gerado fica salvo e acessível no painel.

### 6.1 RESUMO
- 3 formatos: Bullets (5 pontos) | Detalhado (parágrafos) | Por tópicos
- Sempre com citações das fontes
- Botão copiar e exportar

### 6.2 GUIA DE ESTUDO
- Documento completo com tópicos, explicações, conceitos-chave e referências
- Formatado em Markdown bonito
- Exportável como PDF

### 6.3 FLASHCARDS
- IA gera pares pergunta/resposta a partir das fontes
- Interface interativa: card que vira ao clicar (front/back)
- Marcar cada card: Fácil | Médio | Difícil
- Algoritmo de repetição espaçada SM-2 para agendar revisões
- Contador de cards dominados vs pendentes
- Barra de progresso
- Adicionar flashcards manualmente
- Exportar como CSV

### 6.4 QUIZ
- IA gera perguntas com base nas fontes
- Tipos: Múltipla escolha (4 alternativas) | Verdadeiro/Falso | Resposta curta
- Configurar: nº de perguntas (5, 10, 15, 20), dificuldade (fácil, médio, difícil)
- Interface de quiz limpa: uma pergunta por vez, progress bar, timer opcional
- Ao finalizar: score, explicação de cada resposta com citação da fonte
- Histórico de tentativas com evolução

### 6.5 MAPA MENTAL
- IA analisa as fontes e gera estrutura hierárquica de conceitos
- Renderizar com React Flow: nós arrastáveis, zoom, pan
- Nós: tópico principal no centro, subtópicos ramificando
- Cores por nível de profundidade
- Expand/collapse de ramos
- Exportar como PNG

### 6.6 LINHA DO TEMPO (Timeline)
- Se as fontes contêm eventos cronológicos, gerar timeline visual
- Layout vertical com marcadores de data
- Cada evento: data, título, descrição, citação da fonte
- Scroll suave entre eventos

### 6.7 TABELA DE DADOS (Data Table)
- IA extrai informações comparáveis das fontes e organiza em tabela
- Ex: comparação de conceitos, datas, características
- Tabela editável pelo usuário (adicionar/remover linhas e colunas)
- Exportar como CSV ou copiar

### 6.8 AUDIO OVERVIEW (PODCAST) ⭐ FUNCIONALIDADE PRINCIPAL
Replicar EXATAMENTE o comportamento do NotebookLM:

a) Botão "Gerar Audio Overview"
b) A IA primeiro gera um SCRIPT de conversa entre dois apresentadores:

Formatos disponíveis (selecionável pelo usuário):

Player de áudio:

- Pular 10s frente/trás
- Download MP3
- Visualização de forma de onda

MODO INTERATIVO DO PODCAST:

## 7. DEEP RESEARCH (Pesquisa Profunda na Web)

Adicionar na tela de fontes do caderno um botão "Pesquisar na Web":

### Fast Research:
- Input: tema/pergunta
- Busca ~10 fontes web relevantes (usar Serper API ou similar)
- Mostra lista com título, URL, snippet
- Botão "Adicionar ao caderno" em cada resultado
- Resultados viram fontes no caderno

### Deep Research:
- Input: tema/pergunta complexa
- A IA cria um PLANO DE PESQUISA (mostra para o usuário)
- Executa pesquisa autônoma em background:
  - Busca dezenas de fontes na web
  - Lê e analisa cada uma
  - Compila relatório detalhado com citações
- Barra de progresso + estimativa de tempo
- Ao finalizar: mostra relatório + lista de fontes encontradas
- Botão "Adicionar tudo ao caderno"
