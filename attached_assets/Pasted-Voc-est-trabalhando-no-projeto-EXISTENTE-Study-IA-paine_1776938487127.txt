### BACKEND — Como filtrar:

Toda API route de métricas do Admin deve aceitar query params:

  GET /api/admin/metrics?from=2025-07-01T00:00:00Z&to=2025-07-19T23:59:59Z

  GET /api/admin/ia-costs?from=...&to=...
  GET /api/admin/ia-costs/by-provider?from=...&to=...
  GET /api/admin/ia-costs/by-feature?from=...&to=...
  GET /api/admin/ia-costs/by-model?from=...&to=...
  GET /api/admin/ia-costs/by-source?from=...&to=...
  GET /api/admin/users?registeredFrom=...&registeredTo=...&lastAccessFrom=...&lastAccessTo=...
  GET /api/admin/financial?from=...&to=...
  GET /api/admin/logs?from=...&to=...

Se from/to não forem enviados, usar padrão de 30 dias.

### Queries no banco:

Aplicar WHERE em TODAS as queries de métricas:

  WHERE createdAt >= $from AND createdAt <= $to

Ou no caso de daily_metrics:

  WHERE date >= $fromDate AND date <= $toDate
