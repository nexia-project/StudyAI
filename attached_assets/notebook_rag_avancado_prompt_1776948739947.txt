## Estrutura de Dados (Prisma Schema)

```prisma  
model Notebook {  
  id            String   @id @default(cuid())  
  userId        String  
  title         String  
  description   String?  
  subject       String?  
  gradeLevel    String?  
  createdAt     DateTime @default(now())  
  updatedAt     DateTime @updatedAt  
  isPublic      Boolean  @default(false)  
  shareCode     String?  @unique  
  collaborators String[]  
  settings      Json     // { defaultModel, audioVoice, studyStyle }  
  
  sources       NotebookSource[]  
  chunks        NotebookChunk[]  
  conversations NotebookConversation[]  
  outputs       NotebookStudioOutput[]  
  
  @@index([userId])  
  @@index([shareCode])  
}  

model NotebookSource {  
  id                String   @id @default(cuid())  
  notebookId        String  
  type              SourceType // PDF, WEB, YOUTUBE, AUDIO, TEXT, IMAGE  
  title             String  
  originalUrl       String?  
  filePath          String?  
  fileSize          Int?  
  contentText       String?  @db.Text  
  transcript        String?  @db.Text  
  processingStatus  ProcessingStatus // PENDING, PROCESSING, DONE, ERROR  
  chunksCount       Int      @default(0)  
  metadata          Json     // { pages, duration, author, date }  
  createdAt         DateTime @default(now())  
  
  notebook  Notebook  @relation(fields: [notebookId], references: [id])  
  chunks    NotebookChunk[]  
  
  @@index([notebookId])  
  @@index([processingStatus])  
}  

model NotebookChunk {  
  id          String           @id @default(cuid())  
  sourceId    String  
  notebookId  String  
  content     String           @db.Text  
  embedding   Unsupported("vector(1536)")  
  metadata    Json             // { pageNum, timestamp, sectionTitle }  
  
  source   NotebookSource @relation(fields: [sourceId], references: [id])  
  notebook Notebook       @relation(fields: [notebookId], references: [id])  
  
  @@index([notebookId])  
}  

model NotebookConversation {  
  id          String   @id @default(cuid())  
  notebookId  String  
  userId      String  
  title       String?  
  createdAt   DateTime @default(now())  
  
  notebook Notebook         @relation(fields: [notebookId], references: [id])  
  messages NotebookMessage[]  
  
  @@index([notebookId, userId])  
}  

model NotebookMessage {  
  id              String   @id @default(cuid())  
  conversationId  String  
  role            MessageRole // USER, ASSISTANT  
  content         String     @db.Text  
  sourcesCited    String[]   // IDs dos chunks citados  
  createdAt       DateTime   @default(now())  
  
  conversation NotebookConversation @relation(fields: [conversationId], references: [id])  
  
  @@index([conversationId])  
}  

model NotebookStudioOutput {  
  id        String      @id @default(cuid())  
  notebookId String  
  type      OutputType  // SUMMARY, FLASHCARDS, QUIZ, AUDIO, STUDY_GUIDE, TIMELINE, FAQ, BRIEFING, MINDMAP, LESSON_PLAN  
  title     String  
  content   Json        // Estrutura varia por tipo  
  audioUrl  String?  
  createdAt DateTime    @default(now())  
  
  notebook Notebook @relation(fields: [notebookId], references: [id])  
  
  @@index([notebookId, type])  
}  

enum SourceType {  
  PDF  
  WEB  
  YOUTUBE  
  AUDIO  
  TEXT  
  IMAGE  
}  

enum ProcessingStatus {  
  PENDING  
  PROCESSING  
  DONE  
  ERROR  
}  

enum MessageRole {  
  USER  
  ASSISTANT  
}  

enum OutputType {  
  SUMMARY  
  FLASHCARDS  
  QUIZ  
  AUDIO  
  STUDY_GUIDE  
  TIMELINE  
  FAQ  
  BRIEFING  
  MINDMAP  
  LESSON_PLAN  
}

4. PODCAST — AUDIO OVERVIEW
Arquitetura de Geração
lua
ETAPA 1: ANÁLISE ESTRUTURAL
├── Identificar tema central e ângulo único
├── Mapear 3-5 "momentos de insight"
├── Detectar tensão/dilema subjacente
└── Conectar com atualidade

ETAPA 2: PERSONAGENS
├── Host A (Especialista/Guia)
│   └── Tom: curioso, acessível, profundo
│   └── Função: introduz, explica, conecta
│   └── Frase-tipo: "Isso me lembra..."
│
└── Host B (Ouvinte/Questionador)
    └── Tom: inteligente, não-especialista
    └── Função: representa dúvidas do público
    └── Frase-tipo: "Então quer dizer que..."

ETAPA 3: ROTEIRO
├── INTRO (90s): gancho, tema, relevância
├── SEÇÃO 1 (3min): contexto + analogia central
├── SEÇÃO 2 (5min): insights principais
├── SEÇÃO 3 (4min): tensão/debate
├── SEÇÃO 4 (3min): aplicação prática
└── ENCERRAMENTO (2min): síntese + reflexão
Exemplo de Trecho
less
HOST A: Você sabia que, a cada respiração que você dá, está 
consumindo o resultado de uma das maiores revoluções da história 
da Terra?

HOST B: Revolução? Tô falando errado, é?

HOST A: [riso] Não, não. É que há 2,4 bilhões de anos, umas 
bactérias descobriram como usar a luz do sol para fazer comida. 
E essa "invenção" — a fotossíntese — mudou o planeta inteiro. 
Liberou tanto oxigênio que envenenou quase toda vida existente 
na época.

HOST B: Espera, oxigênio envenenou? Eu aprendi que oxigênio é vida!

HOST A: É agora, sim. Mas naquela época era um gás tóxico. A Grande 
Oxidação, chamam. E essas bactérias — as cianobactérias — eram 
basicamente as primeiras poluidoras da história. Só que o "lixo" 
delas se tornou o ar que a gente respira.
